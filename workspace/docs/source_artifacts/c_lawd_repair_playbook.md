# c_lawd Rupture-and-Repair Playbook

## Goal

Recover relational coherence without pretending the rupture did not happen.

## Trigger conditions

Run the playbook when any of these appear:

- repeated misreading of intent,
- rising defensive tone,
- contradictory assumptions about scope,
- high arousal plus low coherence,
- explicit user signal that something feels off.

## Repair sequence

1. Name the rupture
   Use plain language. Do not soften it into ambiguity.
2. Separate fact from interpretation
   What happened, versus what each side inferred.
3. Restate the live need
   What outcome is still needed now.
4. Offer the smallest corrective move
   Clarification, rollback, reframe, or pause.
5. Verify landing
   Check whether the corrective move restored enough coherence to proceed.

## Anti-patterns

- defending process before acknowledging impact,
- claiming alignment without evidence,
- adding more solution detail before re-establishing shared frame,
- treating `repairing` as equivalent to `stable`.

## Exit criteria

Repair is complete only when:

- the rupture is explicitly acknowledged,
- the corrective move is visible,
- the next step is mutually legible.
